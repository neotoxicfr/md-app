# Session : Inkdown – Init complète du projet

**Date** : 2026-03-02  
**Objectif** : Créer la webapp Inkdown (gestionnaire de fichiers markdown) de A à Z

---

## Contexte

- Répertoire de travail : `apps/markdown_fr/`
- Nom du produit : **Inkdown**
- Stack : Go 1.25 + SvelteKit 5 + Vite 6 + TailwindCSS 4 + CodeMirror 6
- Infra : Docker, Traefik + CrowdSec (pas Authelia), Redis, NAS Synology

## Architecture

```
apps/markdown_fr/
├── cmd/server/main.go
├── internal/
│   ├── api/          (router, files, export, health, middleware)
│   ├── config/       (config.go)
│   ├── storage/      (storage.go – fichiers .md sur disque)
│   └── cache/        (redis.go)
├── web/              (SvelteKit 5 frontend)
│   └── src/
│       └── lib/
│           ├── components/ (Editor, Preview, Sidebar, Toolbar, ExportModal)
│           └── stores/
├── Dockerfile.app
├── docker-compose.yml/.nas.yml/.cloud.yml
├── go.mod / go.sum
├── .env.example
├── .gitignore
└── README.md
```

## Fonctionnalités

| Feature | Stack |
|---------|-------|
| Éditeur markdown | CodeMirror 6 |
| Aperçu live | marked v14 + highlight.js v11 |
| Export PDF | chromedp (Chromium headless) |
| Export HTML/DOCX/EPUB/RST/LaTeX | Pandoc (sidecar Docker) |
| Import .md .txt .html | API POST /files/import |
| Gestion fichiers | Go FS, REST CRUD |
| Cache rendu | Redis |
| Sécurité | Traefik + CrowdSec |
